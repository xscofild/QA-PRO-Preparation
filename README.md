# 🧠 QA-PRO-Preparation

Interactive interview-preparation dashboard for **Junior QA Engineer** roles — built as a single-page HTML app with flashcards, quizzes, cheat sheets, project stories, market notes, and a 4-week revision plan.

> Personal QA interview cockpit: Manual QA, API, Java, Selenium, SQL, Git, CI/CD, Onyx project preparation, and ready-to-use interview scripts in RU/EN/DE.

## 🔑 Key Features

### Interactive dashboard
The app is not a static note file. It works as a browser-based study tool with navigation tabs, progress tracking, flashcards, quiz mode, and topic-based sections.

### QA interview focus
Content is organized around real interview preparation, not theory for theory's sake:

- Manual QA fundamentals
- Bug reports and test design
- API testing and HTTP status codes
- Java basics for QA automation
- Selenium WebDriver and locators
- SQL and database checks
- Git, CI/CD, Docker, BDD, mobile testing basics

### ONYX project preparation
Includes a dedicated section for explaining the **Onyx TaskTracker QA project** in interview format:

- backend/frontend/QA architecture overview
- API + UI automation contribution
- JWT/cookies/API flow understanding
- STAR-style project story
- improvement points and realistic next steps

### Cheat Code answer formula
The project includes a simple answer framework for interviews:

1. Give a short definition
2. Explain how it works
3. Give a QA example
4. Mention common mistakes
5. Connect it to real project experience

This helps avoid long, messy answers during interviews.

### Multi-language interview scripts
Ready-to-use answers are included in:

- **RU** — for preparation and self-explanation
- **EN** — for international companies
- **DE** — for German HR/interview situations

## 🚀 How to Run

### Option 1: Open locally
Download the repository and open `index.html` in any modern browser.

```bash
git clone https://github.com/xscofild/QA-PRO-Preparation.git
```

```bash
cd QA-PRO-Preparation
```

Then open:

```bash
index.html
```

### Option 2: Run with VS Code Live Server
1. Open the project in VS Code
2. Install the **Live Server** extension
3. Right-click `index.html`
4. Select **Open with Live Server**

### Option 3: Publish with GitHub Pages
1. Open repository settings
2. Go to **Pages**
3. Select branch: `main`
4. Select folder: `/root`
5. Save

After deployment, the dashboard can be opened directly in the browser.

## 📝 Notes

- This is a **personal learning and interview-preparation project**, not a production SaaS application.
- The project is intentionally kept as a single `index.html` file to make it easy to open, move, publish, and maintain.
- The content is focused on **Junior QA Engineer** roles in Germany/EU with emphasis on API testing, UI automation, Java, Selenium, REST Assured, SQL, and practical interview readiness.
- Future improvements could include splitting the app into separate files, adding JSON-based question storage, search by keyword, and exportable progress reports.
